# セキュリティポリシー (SECURITY.md) - MiSysMon

本文書は、`MiSysMon` プロジェクトにおけるセキュリティ方針および脆弱性報告手順をまとめたドキュメントです。

---

## 1. セキュリティ設計と安心性

`MiSysMon` は以下の設計原則により、高いセキュリティと信頼性を保証しています。

1. **メモリ安全性**:
   - Rust 言語の強力な所有権システムと型安全性を活用し、メモリ安全性上の脆弱性（バッファオーバーフロー、ヌルポインタ参照等）を根本的に排除します。
2. **サンドボックスと権限管理**:
   - 原則としてローカルの読み取り権限のみで動作します。温度取得のために一部の環境で管理者権限が必要となる場合がありますが、ネットワーク通信を一切行わないため、外部との接続を狙った攻撃への露出はありません。
3. **依存ライブラリの自動監査**:
   - Dependabot を通じて、使用している依存パッケージ（`sysinfo` や `eframe` など）の脆弱性を毎週監視し、最新の修正パッケージへの追従を図っています。

---

## 2. サポートされるバージョン

以下のバージョンに対してセキュリティアップデートが提供されます：

| バージョン               | サポート状況 |
| :----------------------- | :----------: |
| 最新リリース (`v1.x.x`)  | ✅ サポート  |
| 古いリリース             | ❌ 非サポート |

---

## 3. 脆弱性の報告方法

`MiSysMon` に潜在的なセキュリティ脆弱性を発見された場合は、公開 Issue を作成するのではなく、以下の手順でご報告ください。

1. **報告窓口**:
   - リポジトリの管理者に直接ご連絡いただくか、セキュリティ報告用窓口へメールを送信してください。
2. **記載いただきたい内容**:
   - 影響を受ける `MiSysMon` のバージョンとOS環境
   - 脆弱性の詳細と再現手順（再現コードまたはコマンド例）
3. **対応プロセス**:
   - 報告を受領後、3日以内に受領確認を行い、修正パッチを作成・検証したうえで速やかにリリースを行います。
