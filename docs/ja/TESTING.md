# テスト方針・実行ガイド (TESTING.md) - MiSysMon

本文書は、`MiSysMon` プロジェクトにおけるユニットテスト、結合テスト、および品質検証に関する方針と実行手順をまとめたドキュメントです。

---

## 1. テスト概要

`MiSysMon` は Rust, egui, eframe, sysinfo 等を使用した軽量なシステムモニターであり、信頼性と堅牢性を維持するために検証を行います。

主要なテスト観点：
- **メトリクス取得**: CPU使用率・温度、メモリ使用率、ネットワーク帯域、ディスク使用量、ディスクI/Oの取得・算出精度
- **二重起動防止**: 共有ライブラリ `common_lib` を通じた Named Mutex による二重起動制限の挙動
- **座標保存と境界検証**: ウィンドウ位置の RON 保存・ロード、および起動時の座標がモニター表示範囲内かどうかの検証 (Win32 API `MonitorFromPoint` 経由)
- **UIレイアウトと挙動**: ドラッググリップでの画面絶対座標アンカー移動、画面端 15px 内でのオートスナップ（吸着）、トグル設定によるウィンドウ幅の動的リサイズ

---

## 2. テストの実行手順

ローカル環境でのテスト実行手順は以下の通りです。

### ユニットテストの実行
```bash
cargo test
```

### 品質・コード検証コマンド
コード変更完了時には、以下の標準コマンドが全てエラー・警告なしで合格することを確認します：

```bash
# 1. ユニットテスト実行
cargo test

# 2. 静的解析 (Clippy)
cargo clippy --all-targets -- -D warnings

# 3. コードフォーマット検証
cargo fmt --check

# 4. Rustdoc ビルド検証
cargo doc --no-deps --document-private-items
```

---

## 3. テストの記述ガイドライン

1. **モジュール内テスト**:
   - `src/main.rs` 内にユニットテスト（バイトフォーマットなど）を記述し、動作の正当性を確認します。
2. **パニック回避のテスト**:
   - センサー取得の失敗や OS 依存処理に起因してアプリケーション全体がクラッシュしないためのフォールバック処理をテスト・検証します。
