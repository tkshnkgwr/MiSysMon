# Specification: Mini System Monitor

**バージョン**: 0.1.16
**内部バージョン**: 0.1.16.0

## 1. 概要
低リソース環境（Windows）で動作する、極限まで軽量なシステムモニタ。
常に最前面に表示され、システムリソースをリアルタイム（1秒間隔）で監視する。

## 2. 技術スタック
- **Prototype:** React, Tailwind CSS, Framer Motion
- **Production:** Rust, egui, eframe (0.35+), sysinfo (0.38+), chrono, serde, serde_json, common_lib (ローカル共有クレート)

## 3. 外観仕様
- **サイズ:** 1100px x 32px (横長バー)
- **装飾:** タイトルバー非表示 (Borderless)
- **背景:** 半透明 (Alpha: 200/255)
- **フォント:** 視認性の高いサンセリフ
- **配置:** 常に最前面 (Always on Top)

## 4. 機能仕様
- **CPU監視:** `sysinfo` を使用し、全コアの平均使用率を表示。
- **CPU温度:** センサーから温度を取得し表示。
- **メモリ監視:** 使用済みメモリ / 全メモリ のパーセンテージを表示。
- **ネットワーク監視:** 合計アップロード/ダウンロード速度を表示（上り `^` / 下り `v`）。
- **ディスク容量監視:** システムディスクの使用量と全容量を表示 (例: 123GB/512GB)。
- **ディスクI/O監視:** 全プロセスのディスク使用量を合算して算出した、リアルタイムの読み込み (R) / 書き込み (W) 速度を表示.
- **バージョン表示:** IO表示と時計表示の間にアプリケーションのバージョン (`vX.Y.Z` 形式) を表示。
- **時計表示:** 右端に現在日時を `yyyy/mm/dd(Day) hh:mm:ss` 形式で表示。
- **移動:** 左端のグリップアイコン `=` をドラッグしてウィンドウを移動。
- **終了:** 右端の「×」ボタンでアプリケーションを終了。
- **位置保存:** 終了時のウィンドウ位置を自動保存し、次回起動時に復元。マルチモニター構成の変更などにより保存位置が現在の画面外となった場合は、自動的に検知してデフォルト位置（画面中央）にリセットして表示する。
- **二重起動防止**: 共有ライブラリ `common_lib`（Windows の名前付きミューテックスの安全なラッパー）を利用した多重起動防止機能。既にアプリが起動している場合は、新しく起動されたプロセスが即座に正常終了する。
- **ログローテーション:** 起動時に温度センサー取得のデバッグログ `sensors_debug.log` を最大3世代までローテーションし（`.1`, `.2`, `.3`）、ディスク消費量を一定に抑える。また、センサーが検出できない場合は、ログに管理者権限の起動を推奨する旨の注記を書き出す。
- **ウィンドウのオートスナップ（吸着）**: ウィンドウを画面端（上下左右の境界線）の 15 ピクセル以内にドラッグした際、端に自動的に吸着して配置が容易になる機能。設定画面からオン・オフを切り替え可能。
- **ミニトレンドライン表示**: CPUとメモリ使用率の隣に、直近 30 秒間の履歴を高さ 12px のスパークライン（折れ線グラフ）としてリアルタイム描画する機能。
- **設定カスタマイズ画面**: ドラッグハンドル（`=`）のダブルクリック、またはメインパネル全体の右クリックで設定用の別ウィンドウを最前面に表示。更新間隔（0.5s〜5.0s）、背景の不透明度（0.3〜1.0）、ウィンドウ吸着の有無、および各表示項目のトグル（CPU、メモリ、ネットワーク、ディスク、IO、バージョン、時計）が変更可能。
- **ウィンドウ幅の動的リサイズ**: 表示項目をカスタマイズした際、表示されているコンポーネントの必要幅を自動計算してウィンドウサイズを動的にリサイズし、無駄な余白を排除する機能。
- **視認性:** テキスト選択を無効化し、誤操作を防止。

## 5. 最適化
- **描画更新:** 設定された更新間隔（デフォルト1.0秒、0.5〜5.0秒で可変）に同期してメトリクスを更新。
- **レイアウト安定化:** 各項目に固定幅を割り当て、数値変動によるガタつきを防止。表示コンポーネントが動的に変更された際のみ、ウィンドウ幅を追従リサイズする。
- **API互換性:** `sysinfo` 0.38 および `eframe` 0.35 の最新APIに準拠。
- **最小限のシステムスキャン:** 必要な CPU・メモリ情報のみをリフレッシュ。また、プロセス情報更新においてもディスク使用量以外の無駄なデータ取得をスキップ。
