# Todo List: Mini System Monitor (MiSysMon)

本プロジェクトの開発ステータス、直近のタスク、および将来的なバックログ（機能拡張のロードマップ）を管理するドキュメントです。

---

## 1. 実装済みの機能 (Done)

- [x] **基本UIフレームワークの構築**:
  - `egui` および `eframe` を使用したスリムな横長バー (1100px x 32px) のデスクトップウィンドウ表示。
  - タイトルバー非表示 (Borderless)、背景透過 (Alpha: 200/255)、常に最前面 (Always on Top)、およびリサイズ禁止の設定。
- [x] **リアルタイム・システムメトリクス取得**:
  - **CPU**: `sysinfo` を用いた全コアの平均使用率のパーセンテージ取得。
  - **CPU温度**: センサー一覧から「CPU」「Core」「Package」などのラベル付きセンサーを検知して温度を取得。検知できない場合は全センサーの最高温度を採用するフォールバック処理。
  - **メモリ**: 使用済みメモリ割合のパーセンテージ取得。
  - **ネットワーク**: インターフェースごとの受信・送信トラフィックを合算し、前回の更新との差分からアップロード/ダウンロード速度を算出して表示。
  - **ディスク**: 全ディスクの合計空き容量・総容量の表示 (GB単位)。
  - **ディスクI/O**: 全プロセスのディスク読み込み・書き込みバイト数を合算し、現在のI/O速度（R/W）を算出して表示。
- [x] **診断・デバッグ用機能**:
  - 起動時に検出されたすべての温度センサー情報（ラベルと温度）を `sensors_debug.log` に書き出す診断ログ出力機能。
- [x] **ユーティリティ・利便性機能**:
  - 左端のドラッググリップ（`=`）を用いた、タイトルバーなしウィンドウのドラッグ移動。
  - 終了時のウィンドウの左上座標を自動で JSON 永続化保存し、次回起動時にその座標に復元する機能。
  - 右端の終了ボタン（`×`）の設置。
  - 右端固定の時計表示（点滅しない固定幅レイアウト `yyyy/mm/dd(Day) hh:mm:ss`）。
  - テキスト選択の無効化（誤クリック・ドラッグによるガタつき防止）。
- [x] **多重起動防止機能**:
  - 共有ライブラリ `common_lib` の名前付きMutexによる起動制御。すでに同一のキーを持つインスタンスが動いている場合は、新しいプロセスは何もせず即終了する。

## 2. 直近のタスク (In Progress / Todo)

- [ ] **ネットワークおよびディスクI/O速度の初回描画時バグ修正**
  - *現象*: 起動時の最初の更新（約1秒後）において、`prev_net_up` と `prev_net_down` が初期値 `0` であるのに対し、`networks.refresh(true)` で取得した「起動から現在までの累積通信量」がそのまま差分として計算される。このため、起動直後の1秒間だけ通信速度が極端に巨大な数値（数GB/sなど）になって表示される。
  - *対策*: アプリケーションの初期化処理 (`SystemMonitor::new`) の段階で、一度 `networks.refresh(true)` を呼び出し、現在の累積値を `prev_net_up` および `prev_net_down` に初期セットしておく。
- [ ] **マルチモニター環境におけるウィンドウ位置復元のロバスト性向上**
  - *課題*: ディスプレイの切断や解像度変更時に、保存された座標が画面外になってしまう可能性がある。
  - *対策*: 復元する座標が、現在のディスプレイの有効範囲内（スクリーン座標内）にあるかどうかを検証し、画面外の場合はデフォルト位置（画面上部中央など）に配置するフォールバックロジックを追加する。
- [ ] **温度センサー取得処理のクリーンアップ**
  - *課題*: Windowsのセキュリティ仕様により、管理者権限で実行しないと温度センサー情報が取得できないケースが多い。
  - *対策*: 温度取得失敗時（`0.0` の場合）のUI表示を `--°C` にするだけでなく、起動時の `sensors_debug.log` に「温度取得には管理者権限が必要な可能性がある」旨の注意書きを追加、あるいはドキュメントへの明記を行う。

## 3. 将来の拡張・バックログ (Backlog)

- [ ] **設定カスタマイズ画面の追加**
  - 右クリックや特定のキーバインド（またはグリップ部のダブルクリック）で、設定ポップアップを表示。
  - 更新間隔（現在は1秒固定）の変更機能（0.5秒〜5秒など）。
  - ウィンドウの透過度（不透明度）の調節スライダー。
  - 表示項目のオンオフ（CPUは要るがディスクIOは非表示にしたい等、個別にトグルできる機能）。
- [ ] **ウィンドウのオートスナップ（吸着）機能**
  - ウィンドウをデスクトップの画面端（特に上下左右の境界線）にドラッグした際、端にピタッと吸着して配置が容易になる機能。
- [ ] **リソース使用率のグラフ（ミニトレンドライン）表示**
  - 各項目の文字列の隣や背景に、直近10〜30秒間のCPU・メモリ使用率の推移を10px程度の高さのミニスパークライン（簡易折れ線グラフ）として描画する機能。
- [ ] **ログファイルのローテーション機能**
  - `sensors_debug.log` の出力において、エラー履歴やパフォーマンス記録などを保持・ローテーションし、ディスク書き込み量を最小化する。
