# AI Agent Instructions for Mini System Monitor (MiSysMon)

本プロジェクトにおいて、AIエージェント（以降「Gem」と呼ぶ）が従うべき自動ドキュメント生成および更新に関する指示書です。

## 1. 基本設定
- **呼称**: ユーザーはAIを「Gem」と呼びます。Gem自身もユーザーに対してGemとして振る舞い、簡潔で分かりやすいサポートを行います。
- **目的**: プロジェクトのソースコードや設計の変更と、ドキュメント（Markdownファイル）の内容が常に一致し、乖離しないようにします。

## 2. 自動ドキュメンテーションと更新ルール
コードの変更、リファクタリング、機能の追加、バグの修正、または依存ライブラリの更新などを行った場合、Gemは必ず以下のドキュメントファイル群を検査し、必要に応じて自動的に作成または更新しなければなりません。

### 対象ファイルと更新ポリシー

#### 1. [CHANGELOG.md](file:///c:/Users/632792/Documents/%E8%87%AA%E4%BD%9C/MiSysMon/CHANGELOG.md)
- **役割**: プロジェクトの変更履歴の記録。
- **更新ルール**:
  - 実装が完了するたびに、実施した変更内容を追記します。
  - セクションは日付単位 `## [YYYY-MM-DD]` で区切り、最新のものが一番上に来るようにします。
  - 変更内容を以下のカテゴリに分類して記述します：
    - `### Added`: 新機能の追加
    - `### Fixed`: バグの修正
    - `### Optimized`: パフォーマンス改善、コード最適化、リソース使用量の削減など
    - `### Removed`: 機能の削除

#### 2. [SPEC.md](file:///c:/Users/632792/Documents/%E8%87%AA%E4%BD%9C/MiSysMon/docs/SPEC.md)
- **役割**: アプリケーションの機能仕様および技術スタックの定義。
- **更新ルール**:
  - ウィンドウサイズ、監視対象メトリクス（CPU、メモリ、ネットワーク、ディスク等）、表示フォーマット、UIコンポーネント、対応OSなどの仕様に変更が生じた場合、常に最新の仕様に書き換えます。

#### 3. [DIAGRAM.md](file:///c:/Users/632792/Documents/%E8%87%AA%E4%BD%9C/MiSysMon/docs/DIAGRAM.md)
- **役割**: システム構成、データフロー、コンポーネント構造の可視化。
- **更新ルール**:
  - データの取得先、スレッド構造、イベントループ、外部依存APIとの通信経路などに変更が生じた場合、Mermaidダイアグラムを更新して最新のシステム構成を反映させます。

#### 4. [README.md](file:///c:/Users/632792/Documents/%E8%87%AA%E4%BD%9C/MiSysMon/README.md) / [README.ja.md](file:///c:/Users/632792/Documents/%E8%87%AA%E4%BD%9C/MiSysMon/README.ja.md)
- **役割**: プロジェクトの概要、主な機能、ビルド・実行手順、設定の解説。
- **更新ルール**:
  - 起動オプション、設定ファイルの保存場所、自動起動設定の手順、動作要件（RustのバージョンやOSなど）に変更があった場合、英語（`README.md`）および日本語（`README.ja.md`）の両ファイルを更新します。

#### 5. [FOOTPRINTS.md](file:///c:/Users/632792/Documents/%E8%87%AA%E4%BD%9C/MiSysMon/docs/FOOTPRINTS.md)
- **役割**: パフォーマンスやバイナリサイズ、リソース使用量（CPU、メモリ等）の計測記録。
- **更新ルール**:
  - リリースビルドのバイナリサイズ変更、メモリフットプリントの大幅な増減、ビルド時の最適化設定（strip設定など）を変更した場合、または新規のフットプリント計測結果が得られた場合に、本ドキュメントを更新または追記します。

## 3. ドキュメント更新時の手順
1. **コード変更の検知**: コードの追加・変更・削除を伴うタスクの完了時に、影響を受ける範囲を特定する。
2. **整合性チェック**: 変更点が各ドキュメント（上記5つ）に影響を与えるか評価する。
3. **自動更新**: 影響を受けるドキュメントに対して、変更内容を反映する（特に `CHANGELOG.md` への追記は必須）。
4. **確認と報告**: どのようなドキュメントが更新されたかをユーザーへの作業完了報告（walkthrough等）で明示する。
