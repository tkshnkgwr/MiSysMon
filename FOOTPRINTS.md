# Footprints: Mini System Monitor Development Journey

本プロジェクトの開発における技術的変遷、意思決定の背景、および修正の全記録です。

## 🐾 MileStone 1: Prototype Phase (React -> Rust への決断)
- **課題:** React/Vite プロトタイプでは Windows の低リソース環境においてブラウザエンジンのオーバーヘッドが無視できない。
- **決定:** 極限の低負荷を実現するため、Webview を介さない Pure Rust (egui/eframe) への完全移植を決定。
- **成果:** メモリ消費量を数十MB単位から数MB単位へ、CPU負荷をほぼゼロ％へ削減。

## 🐾 MileStone 2: Rust 安定化と API 追従
- **技術的障壁:** `sysinfo` クレートのバージョン 0.30 へのメジャーアップデートに伴い、データ取得方法が大幅に変更。
- **修正内容:** `SystemExt` などの古いトレイトを廃止し、新しい `refresh_all()` およびコンポーネント取得フローに書き換え。
- **成果:** 最新の依存関係で安定してビルド可能なコードベースの構築。

## 🐾 MileStone 3: UI/UX の洗練と「文字化け」対策
- **課題:** 低リソース環境のフォント不足により、特殊記号（↑, ↓, ☷）が「豆腐（文字化け）」になる。
- **修正内容:** 記号を標準テキスト（^, v, =）に置換。
- **改善:** 各項目に `add_sized` で固定幅を適用し、数値の桁数変動によるバー全体の「ガタつき」を解消。

## 🐾 MileStone 4: 永続化とウィンドウ挙動の制御
- **機能追加:** `serde` を導入し、ウィンドウ位置を JSON 保存する機能を実装。
- **難題:** 再起動時にバーの高さが勝手に 2～3 倍に広がる怪現象が発生。
- **解決策:** `NativeOptions` で `min_inner_size` / `max_inner_size` を完全に同一値に固定し、さらに起動コードで `InnerSize` コマンドを強制実行することで封じ込めに成功。

## 🐾 MileStone 5: Windows 完全対応とリリース最適化
- **課題:** デバッグビルド時や標準設定での実行時に、背後に不要な「DOS窓」が表示される。
- **修正内容:** `#![windows_subsystem = "windows"]` 属性を追加し、GUI専用アプリとして定義。
- **最適化:** `Cargo.toml` に `strip = true` や `lto = true` を追加し、バイナリサイズを最小化。

## 🐾 MileStone 6: 高度な監視項目の追加 (Disk I/O)
- **課題:** `sysinfo` は物理ディスクごとのリアルタイム I/O 速度を直接提供しない。
- **解決:** 全プロセスの `disk_usage` を合算し、前秒との差分を取る計算ロジックを自作。ネットワーク速度も同様に「累計」から「秒間速度」への表示へ変換。

## 🐾 MileStone 7: CPU温度取得の堅牢化 (2026-05-07)
- **課題:** 特殊なセンサーラベル（Tdie, Tctlなど）を持つ環境や、ラベルが未定義の環境で CPU 温度が常時 0.0 と表示される。
- **改善 1:** 従来のラベル判定（CPU/CORE/PACKAGE）に加え、AMD 系の `TCTL`, `TDIE` や `THM` (Thermal) を検索対象に追加。
- **改善 2:** それらが見つからない場合は「システム内の最高温度センサー」の値を CPU 温度として採用。
- **UI 調整:** 取得失敗時、0.0°C と表示されると紛らわしいため、`--°C` と表示されるように変更。
- **診断機能の追加:** 起動時、実行ファイルと同じフォルダに `sensors_debug.log` を書き出すように実装。
- **最終結論:** `Detected Sensors Count: 0` の場合、使用しているライブラリ (`sysinfo`) が Windows 標準 API (WMI) 経由でセンサー情報を取得できない環境であることを特定。
- **設計判断:** リング0ドライバを同梱して配布するのは「低リソース・安全・軽量」という原則に反するため、取得不可時は `--°C` と表示する仕様を最終決定とした。

## 🐾 MileStone 8: ディスク I/O バグ修正とシステム更新の最適化 (2026-06-24)
- **課題:** ディスク I/O 速度が起動直後しか表示されず、2秒目以降に 0 に張り付いてしまうバグが発生していた。また、毎秒の `refresh_all` によりシステム情報全体を走査するため、常駐アプリとして CPU 負荷が不要に発生していた。
- **原因:** `sysinfo` の `Process::disk_usage()` が返す値自体がすでに「前回の更新からの差分」であったが、自前でさらに前回の合計値との差分をとる二重差分計算を行っていた。
- **修正内容:** 自前の差分計算（`prev_disk_read` などの変数）を完全に削除し、合計値をそのまま表示。また、`self.sys.refresh_all()` を廃止し、必要な CPU/メモリの更新のみに絞り、プロセス更新も `ProcessRefreshKind::new().with_disk_usage()` でディスク情報に限定した。
- **成果:** 正確なディスク I/O 速度が表示されるようになり、無駄なシステムスキャンがなくなったことで CPU 負荷がさらに激減し、超軽量動作が実現された。

## 📈 技術的注釈 (FOOTPRINT Logs)

### 2026-04-16: クレート解決のイタチごっこ
- **現象:** `extern crate` の有無でビルドエラーがループ。
- **Why:** モダンな Rust (Edition 2021) では不要なはずだが、ローカル環境のパースエラーにより混乱が生じていた。
- **Fix:** 標準的な `use` のみに統一し、`Cargo.toml` でのバージョン明示により完全解決。

### 2026-04-16: ウィンドウサイズ肥大化の謎
- **現象:** `Pos2` の復元時にサイズが狂う。
- **Why:** eframe 側の自動スケールと手動復元が競合。
- **Fix:** 手動での強制リサイズ命令を追加。

### 2026-04-21: GitHub README 整備
- **意図:** Tauri よりも Rust 純正の方が「軽い」という差別化ポイントを明確化。
