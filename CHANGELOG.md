# Changelog

## [1.0.0] - 2026-07-17
### Added
- **ウィンドウのオートスナップ（吸着）機能**: ドラッグ移動時にデスクトップ作業領域（タスクバー除く）の境界線15px以内に近づくと、自動的にピタッと吸着する機能を Win32 API を用いて実装。
- **リソース使用率のグラフ（ミニトレンドライン）表示**: CPU・メモリ使用率の隣に、過去30秒間の履歴を高さ12pxのミニ折れ線グラフ（シアン色）としてリアルタイム描画する機能を追加。
- **設定カスタマイズ画面（別Viewport）の追加**: グリップ `=` のダブルクリックまたはメイン領域の右クリックで、別ウインドウの設定ポップアップを表示。更新間隔（0.5s〜5.0s）、透過度、スナップの有効化、各項目のトグルが変更可能。
- **ウィンドウ幅の動的リサイズ機能**: 設定で表示項目をオン/オフした際、コンテンツの必要幅に合わせてメインウィンドウのサイズを動的にリサイズし、余白のないコンパクトなレイアウトを維持する機能を実装。
- **AI事前検証プロセスのMarkdown例外追記**: AIエージェント（大賢者）がMarkdownファイル（`.md`）のみを修正した際、ローカルの事前検証プロセス（clippy, fmt, test, doc）をスキップできるルールを `.agents/AGENTS.md` に追加。

### Fixed
- **GitHub Actions ワークフローの修正 (ci.yml / release.yml)**: 未定義の `actions/checkout@v7` および `softprops/action-gh-release@v3` に起因して発生していた GitHub Actions 実行エラーを修正するため、安定版の `checkout@v4` および `action-gh-release@v2` へバージョンを適正化。
- **クロスプラットフォームコンパイル（Clippy）エラーの修正**: Windows以外のターゲット（Linux/macOS）でコンパイルした際、吸着処理用の移動座標変数 `new_pos` が `unused_mut` 警告（エラー扱い）を引き起こしていた不具合に対し、`#[allow(unused_mut)]` を追加してビルドが成功するように修正。

### Removed
- **自動バージョンバンプ機能の廃止**: GitHubでの不要な自動コミットやコンフリクトを回避するため、`Auto Bump Version` ワークフロー (`bump-version.yml`) を削除し、開発者が手動でバージョンバンプおよびタグ管理を行う運用に変更。

## [2026-06-24]
### Fixed
- **ディスクI/O速度の算出バグ修正:** 各プロセスの `read_bytes` / `written_bytes` は `sysinfo` 側ですでに差分（前回の更新以降のバイト数）として返されているため、自前での二重差分計算（`prev_disk_read` 等の引き算）を行っていたバグを修正。これによりディスクI/Oの表示が正しく更新されるようになりました。

### Optimized
- **`sysinfo` 更新処理の軽量化による CPU 負荷削減:**
  - 1秒間隔の `refresh_all()` を廃止し、必要な `refresh_cpu()` および `refresh_memory()` のみの呼び出しに変更。
  - ディスクI/Oを計算するためのプロセス情報更新において、`ProcessRefreshKind::new().with_disk_usage()` を指定することで、余分な情報（CPU、メモリ、環境変数など）の取得をスキップし、ディスク使用量のみを更新するように最適化。
  - アプリ起動時の初期化処理を `System::new()` に変更し、起動時のオーバーヘッドを最小限化。
  - **診断ログ `sensors_debug.log` の出力先変更による安全性向上:** カレントディレクトリが汚れるのを防ぎ、スタートアップ等での起動時の書き込み権限エラーを避けるため、出力先を設定ファイルと同じ `%APPDATA%\Mini System Monitor` ディレクトリに変更しました。
